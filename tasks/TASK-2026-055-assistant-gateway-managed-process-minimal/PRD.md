# PRD

## Background

The earlier gateway settings work migrated the Assistant UI from `channel serve` terminology to `gateway serve`, but the user still has to leave the app and run a manual long-lived CLI command. For local desktop usage, that is the wrong default interaction model.

## Objective

Users can launch and stop a local managed gateway process from the Assistant settings page, and the page can show whether the managed process is currently running or has stopped with an error.
